# ขายดี Stock — Feature Grid Redesign Design

**วันที่:** 2026-07-14
**สถานะ:** อนุมัติทิศทางดีไซน์แล้ว รอผู้ใช้ตรวจสเปกก่อนวางแผนลงมือ

## ปัญหาปัจจุบัน

ส่วน “ฟีเจอร์ที่เชื่อมหน้าร้านกับคลังจริง” ใช้กริดแบบ Bento ซึ่งทำให้การ์ด “คลังสินค้าครบวงจร” และ “รายงานที่ใช้ตัดสินใจ” กินพื้นที่สองคอลัมน์ ขณะเดียวกันทุกการ์ดถูกกำหนดความสูงขั้นต่ำ 256px และใช้กล่องภายในสูงเต็มการ์ด การ์ดขนาดใหญ่จึงมีพื้นที่ว่างมากและดันไอคอนกับข้อความไปอยู่คนละมุมด้านล่าง ลำดับการอ่านไม่สม่ำเสมอกับการ์ดอื่นและทำให้ส่วนนี้ดูไม่สมดุล

## เป้าหมาย

- ทำให้ฟีเจอร์ทั้งหกรายการอ่านและเปรียบเทียบได้เร็ว
- ลดพื้นที่ว่างที่ไม่สื่อความหมาย
- สร้างจังหวะการวางองค์ประกอบที่สม่ำเสมอและดูเป็นมืออาชีพ
- รักษาเนื้อหา ไอคอน สี และภาษาภาพเดิมของหน้า Landing
- รองรับ Desktop, Tablet และ Mobile โดยไม่เกิด horizontal scroll หรือข้อความถูกตัด

## เลย์เอาต์ที่อนุมัติ

เปลี่ยนจาก Bento เป็นกริดการ์ดขนาดเท่ากัน:

- Desktop ตั้งแต่ breakpoint `lg`: 3 คอลัมน์ × 2 แถว
- Tablet ตั้งแต่ breakpoint `md`: 2 คอลัมน์ × 3 แถว
- Mobile: 1 คอลัมน์ × 6 แถว
- ไม่มีการ์ดที่กินพื้นที่มากกว่าหนึ่งคอลัมน์
- การ์ดในแถวเดียวกันสูงเท่ากันตามพฤติกรรมของ CSS Grid แต่ไม่กำหนด `min-height: 256px`
- บน Mobile ความสูงปรับตามเนื้อหาเพื่อไม่ให้เกิดพื้นที่ว่างเกินจำเป็น

## โครงสร้างการ์ด

ทุกการ์ดใช้โครงสร้างและลำดับเดียวกัน:

1. ไอคอน SVG ขนาดและพื้นหลังเดียวกับระบบเดิม วางมุมซ้ายบน
2. หัวข้อฟีเจอร์วางใต้ไอคอน
3. คำอธิบายวางใต้หัวข้อและปล่อยให้ตัดบรรทัดตามธรรมชาติ

องค์ประกอบทั้งหมดชิดด้านบน ไม่ใช้ `margin-top: auto`, `justify-content: space-between` หรือการจัดแนวที่ดันข้อความไปด้านล่าง การ์ดใช้ระยะภายในตามระบบ 8px โดยตั้งเป้า 24px บน Mobile และ 28px ตั้งแต่ Tablet ขึ้นไป ช่องว่างระหว่างไอคอน หัวข้อ และคำอธิบายต้องสม่ำเสมอทุกใบ

## ภาพลักษณ์และ Interaction

- คงพื้นการ์ดสีขาวอมเทา ขอบบาง มุมโค้ง และเงานุ่มจาก `.lp-card`
- คงสีข้อความ ไอคอน และ semantic tokens เดิมเพื่อไม่ให้กระทบส่วนอื่นของหน้า
- ไม่เพิ่มข้อมูล ตัวเลข ภาพประกอบ หรือคำกล่าวอ้างใหม่
- การ์ดเป็นเนื้อหาแบบ static ไม่ทำให้ดูเหมือนปุ่มหรือลิงก์
- Hover และ focus-within เปลี่ยนได้เฉพาะขอบ เงา opacity หรือ transform ที่ไม่กระทบ layout
- ไม่เพิ่ม animation สำหรับการเข้าหน้าหรือเอฟเฟกต์ตกแต่งใหม่

## Responsive และ Accessibility

- ตรวจที่ความกว้าง 390px, 768px และ 1440px
- ลำดับ DOM และลำดับการอ่านคงตามข้อมูลใน `FEATURES`
- หัวข้อยังเป็น `h3` ภายใต้หัวข้อส่วน `h2`
- ไอคอนตกแต่งยังคง `aria-hidden="true"`; ชื่อและความหมายทั้งหมดอยู่ในข้อความ
- ข้อความคำอธิบายมีขนาดอย่างน้อย 16px และ contrast ตาม token เดิม
- ไม่บังคับความสูงที่ทำให้ข้อความล้นเมื่อซูมหรือเพิ่มขนาดตัวอักษร
- ไม่มี horizontal scroll และไม่มีการตัดข้อความด้วย ellipsis

## ขอบเขตการเปลี่ยนแปลง

- แก้เฉพาะ `FeatureGrid` ใน `components/landing/LandingSections.tsx`
- ปรับ regression test ที่เกี่ยวข้องใน `test/landing-source.test.ts`
- เพิ่ม CSS เฉพาะส่วนนี้เมื่อจำเป็นเท่านั้น; ไม่เปลี่ยน `.lp-card` แบบ shared หาก utility classes แก้ปัญหาได้
- ไม่แก้ข้อความใน `components/landing/content.ts`
- ไม่เปลี่ยนส่วน Hero, Product Showcase, Pricing, FAQ หรือระบบหลังบ้าน

## การทดสอบและเกณฑ์ยอมรับ

- Source regression test ยืนยันว่า Desktop ใช้ 3 คอลัมน์และไม่มี `col-span` สำหรับฟีเจอร์
- Source regression test ยืนยันว่าไม่มี `min-h-64`, `h-full`, `mt-auto` หรือเงื่อนไข `isWide` ใน `FeatureGrid`
- Landing tests, ESLint, TypeScript และ production build ผ่าน
- Browser visual QA ที่ 390px, 768px และ 1440px แสดง 1, 2 และ 3 คอลัมน์ตามลำดับ
- ที่ 1440px การ์ดทั้งหกเรียงเป็นสองแถวและองค์ประกอบทุกใบเริ่มจากด้านบนในแนวเดียวกัน
- องค์ประกอบแรกของทุกการ์ดเริ่มหลังขอบบนตาม padding 24px หรือ 28px และไม่เกิน 32px
- ไม่มีข้อความล้น ตัด หาย หรือเกิด horizontal scroll
- Console ไม่มี warning หรือ error ใหม่จากส่วนนี้

## ขอบเขตที่ไม่ทำ

- ไม่คงรูปแบบ Bento หรือการ์ดขนาดกว้างเป็นพิเศษ
- ไม่เพิ่ม CTA ภายในแต่ละการ์ด
- ไม่เปลี่ยนชุดไอคอนหรือโทนสีของหน้า
- ไม่เพิ่ม dependency หรือ client-side JavaScript
